# Bot 系统优化需求单

## 概述
YiYi 内部 Bot 系统的全面优化计划，涵盖双向通信、连接管理、富文本支持、多媒体处理等方面。

---

## 需求列表

### P0 - 必须完成

#### 1. 钉钉 Event Subscription 双向通信
- **现状**：仅支持 Webhook 推送（Outgoing），无法主动监听用户消息
- **目标**：接入钉钉 Stream 模式（长连接），支持接收 @机器人 消息、群聊消息、单聊消息
- **技术方案**：使用钉钉 Stream API（WebSocket 长连接），替代当前 Webhook 被动接收
- **回复方式**：通过钉钉 OpenAPI 回复消息（支持 Markdown/卡片）
- **配置项**：`app_key`, `app_secret`（从钉钉开放平台获取）

#### 2. 飞书 Event Subscription 双向通信
- **现状**：仅支持 Webhook 推送，Event 回调需要公网地址
- **目标**：接入飞书 WebSocket 长连接模式（Long Connection），无需公网地址
- **技术方案**：使用飞书 WebSocket 模式接收事件，通过飞书 OpenAPI 回复
- **回复方式**：通过飞书 IM API 发送消息（支持富文本/卡片）
- **配置项**：`app_id`, `app_secret`（从飞书开放平台获取）

#### 3. Bot 单独启停控制
- **现状**：`bots_start` 启动所有 Bot，无法单独控制
- **目标**：支持 `bots_start_one(bot_id)` / `bots_stop_one(bot_id)` 单独操作
- **前端**：每个 Bot 卡片上有独立的开关按钮

#### 4. Bot 连接状态监控
- **现状**：前端无法知道 Bot 是否在线/断连/重连中
- **目标**：
  - 后端维护每个 Bot 的连接状态（disconnected/connecting/connected/error）
  - 通过 Tauri 事件实时推送状态变化
  - 前端显示状态指示灯（绿/黄/红）
  - 记录最近一次错误信息

### P1 - 重要

#### 5. 富文本回复支持
- **现状**：所有平台都只发送纯文本
- **目标**：
  - Discord: 支持 Embed 和 Markdown
  - Telegram: 支持 HTML/MarkdownV2 格式
  - 钉钉: 支持 Markdown 卡片
  - 飞书: 支持富文本 (post) 消息
  - QQ: 支持 Markdown（如平台允许）

#### 6. 多媒体消息完整处理
- **现状**：Telegram 图片用 `telegram://file/` 占位未下载；飞书图片需要 token 才能访问
- **目标**：
  - 收到多媒体消息时，实际下载文件到临时目录
  - 将图片传给 Agent（通过 Vision API）
  - 支持 Agent 生成图片/文件并通过 Bot 发送

#### 7. 消息发送重试机制
- **现状**：发送失败直接丢弃
- **目标**：
  - 失败后指数退避重试（最多3次）
  - 重试失败后通知前端
  - 记录发送失败日志

#### 8. API 速率限制处理
- **现状**：无限速保护
- **目标**：
  - Discord: 遵循 rate limit headers
  - QQ: 遵循官方频率限制
  - Telegram: 处理 429 响应
  - 使用令牌桶算法统一限速

### P2 - 锦上添花

#### 9. 新平台接入 - 企业微信（WeCom）
- 自建应用 + Webhook 回调接收消息
- REST API 发送消息（text + markdown）
- access_token 缓存（7200s 有效期，提前 5 分钟刷新）
- 凭证测试连接
- AI 引导式开通向导

#### 10. Bot 消息统计面板
- 各 Bot 的消息收发统计
- 响应时间统计
- 活跃用户统计
- 错误率统计

#### 11. Bot 消息模板系统
- 支持自定义欢迎语
- 支持关键词自动回复（不经过 Agent）
- 支持定时群发
