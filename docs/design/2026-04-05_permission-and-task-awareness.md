# 权限弹窗优化 + 任务感知增强

## 需求来源
用户反馈 2026-04-05

---

## 1. 权限弹窗内联到对话消息中

**现状**: 权限请求以模态弹窗（Toast confirm）显示，视觉粗糙  
**目标**: 权限请求作为特殊消息卡片嵌入对话流，带 Accept/Deny 按钮

### 方案
- 新增消息类型 `permission_request`，渲染为带按钮的卡片
- 后端 permission_gate emit 事件 → 前端注入到当前聊天流
- 用户点按钮 → invoke respond_permission_request → 后端 oneshot 解除阻塞
- 按钮点击后变为已处理状态（✅ 已授权 / ❌ 已拒绝）

---

## 2. 后台任务侧边栏呼吸提醒

**现状**: 任务创建后侧边栏无视觉反馈  
**目标**: 新任务创建时侧边栏对应项呼吸/脉冲动画，吸引用户注意

### 方案
- 监听 `task://created` 事件
- 给新任务项添加 CSS pulse 动画（2-3 秒后消失）
- 如果侧边栏折叠，给侧边栏 toggle 按钮加徽标提示

---

## 3. 跨会话任务感知

**现状**: 
- 当前会话中 Agent 创建了任务，但用户问"刚才那个任务怎么样了"时 Agent 无上下文
- 其他会话完全看不到别的会话创建的任务

**目标**: 任何会话都能查询和引用任务状态

### 方案
- Agent 的 system prompt 中注入当前活跃任务摘要
- 提供 `query_tasks` 工具让 Agent 查询任务列表和状态
- 任务和会话的关联关系记录在 DB（task → session_id）
